## 1. 회원가입 인증 (5종)

회원가입 시 다음 5가지 소셜/본인 인증 중 하나를 통과해야 한다.

- 구글 인증
- 카카오 인증
- 네이버 인증
- 라인 인증
- 휴대폰 인증

인증 성공 시 내부 사이트 회원으로 가입되며, 사용자 데이터베이스에 프로필이 생성된다.

### 회원 DB 필수 필드 (제안)

- user_id (내부 식별자)
- auth_provider (google / kakao / naver / line / phone)
- auth_provider_uid
- email (선택)
- phone (휴대폰 인증 시 필수)
- name (인증 제공자에서 받은 값)
- created_at
- last_login_at
- membership_tier (회원 / 비회원)
- discount_applied (10% 적용 여부)

---

## 2. 비회원 인증 (4종)

비회원으로 사용하려면 다음 4가지 중 하나로 본인 확인을 받아야 한다.

- 구글 인증
- 카카오 인증
- 네이버 인증
- 휴대폰 인증

라인은 비회원 인증에는 포함하지 않는다.

비회원도 정보이용료 결제 후 골프여행 가이드 사용은 가능하지만, 회원가입 10% 할인 혜택은 받지 못한다.

---

## 3. 메인화면 AI 대화 사용 제한

메인화면의 AI 대화창은 두 가지 모드로 구분한다.

### 무료 모드 (인증 불필요)

- 기본 인사·일반 대화만 허용
- 일본 골프 관련 기본 정보(시즌, 인기 지역, 일반 안내 등) 응답 가능
- 구체적 골프장 추천·티타임 검색·예약 안내 차단

### 골프여행 가이드 모드 (인증·결제 필수)

- 골프장 추천, 티타임 검색, 가격 조회, 예약 진행 등 핵심 가치 기능
- 무료 모드에서 사용자가 골프여행 가이드성 질문을 시도하면 회원가입 또는 비회원 인증 유도 메시지 노출
- 인증 통과 + 정보이용료 결제 완료 시 잠금 해제

### 유도 메시지 예시 (UX 초안)

- "이 정보는 회원/비회원 인증 후 이용 가능합니다."
- "회원가입하시면 정보이용료 10% 할인이 적용됩니다."

## 4. 정보이용료 정책

- 기본 정보이용료: 1만원
- 회원가입 사용자: 10% 할인 → 9천원
- 비회원 사용자: 정가 1만원

### 결제 흐름 (초안)

```
사용자 → 골프여행 가이드 사용 시도
  ↓
인증 게이트 (회원/비회원)
  ↓
결제 게이트 (회원 9천 / 비회원 1만)
  ↓
결제 완료 → 가이드 세션 활성화 (사용 범위·기간 정책 별도 정의 필요)
```

### 권장 기본값 (사용자 재확인 대기)

사용자 "확정합니다" 응답 시점에 미정 항목이 별도 지정되지 않아 합리적 기본값을 채택한다. 구현 패킷 작성 전 한 번 더 확인한다.

- 사용 기간: **결제 후 7일 또는 예약 1건 완료까지** (둘 중 먼저 도래)
- 환불 정책: **결제 후 가이드 미사용 상태 한해 24시간 내 100% 환불**, 가이드 1턴 이상 사용 시 환불 불가
- 통화 표기: **KRW 기본 + JPY 병기** (메인 가격은 KRW, 일본 현지 비용 안내 시 JPY 보조 표기)
- 결제 수단: **간편결제 우선** (카카오페이, 토스, 네이버페이) + 카드 결제 보조. 계좌이체는 v3.1로 보류

## 5. 2차 업데이트와의 연결

2차 업데이트의 자연어 상담 에이전트 위에 다음 게이트를 추가한다.

```
손님 채팅 진입
  ↓
[NEW] 의도 분류: 일반 대화 vs 골프여행 가이드 요청
  ↓
일반 대화 → 무료 응답
골프여행 가이드 → 인증 게이트 → 결제 게이트 → 가이드 진행
```
