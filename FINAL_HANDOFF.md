- `api/golf-chat.js` — API 보안·입력 제한
- `docs/jp-golf.html` — 베타 UI 문구
- `vercel.json` — Vercel 함수·루트 경로 설정
- `GATE_B_DECISION.md` — 확정한 호스팅·MVP 결정
- `SECURITY_PROGRESS.md` — 적용된 방어와 잔여 보안 항목
- `planning/` — 인증·결제·관리자 검수·ToS 계획 문서

## 사용자 최종 실행 순서

1. 위 파일과 `git diff`를 직접 검토한다.
2. Vercel 프로젝트를 별도로 연결한다.
3. Vercel 환경변수에 `GEMINI_API_KEY`를 입력한다. 키는 저장소와 `.env`에 넣지 않는다.
4. 스테이징에서 `/`와 `/api/golf-chat`을 실제 응답으로 확인한다.
5. 모바일 UI, 405/400/429/502 응답, 키 미노출을 확인한다.
6. ToS·법률·사업 책임자 승인을 확인한다.
7. 모든 게이트 통과 후에만 사용자가 commit/push/deploy 여부를 결정한다.

## 아직 공개하면 안 되는 이유

- 인증·결제와 유료 가이드 모드가 아직 연결되지 않음
- 분산 환경용 rate limit과 세션 인증이 없음
- 공급사 자동 검색·자동 예약·관리자 검수 큐가 미구현
- ToS 원문·법률 검토와 사업 책임자 승인이 없음
- 실제 스테이징·모바일 QA와 롤백 검증이 없음

따라서 현재 결과물은 **내부 검증 및 사용자 최종 검토용**이며, 공개 수익화 승인이나 실제 자동 예약 승인을 의미하지 않는다.

## 금지 사항

- G: 원본 또는 `Jh-golf` 원본을 수정·이동·삭제하지 않는다.
- 승인 전 크롤링·브라우저 자동화·실제 결제·자동 예약을 시작하지 않는다.
- 이 handoff 단계에서는 commit, push, deploy를 수행하지 않는다.
